# Wai & Watts — 001 MVP

**Status:** Active spec  
**Last updated:** 2026-01-17  
**Audience:** Builder GPT + human reviewer

---

## MVP

### What we’re building
A local-only Spring Boot backend that ingests **public New Zealand datasets** and exposes REST APIs for querying:

- **LAWA** river water quality *state & trend* results (derived assessments, not raw monitoring time series)
- **MBIE** electricity generation statistics (time series by fuel; renewable share derived)
- **MBIE** renewables statistics (annual, flexible metrics)

Core value: a reviewer can run the system locally, ingest real public data, and explore clean APIs + a grounded “overview” endpoint that includes explicit source metadata (“fact pack”).

### MVP user flows
1. Start the system locally (Postgres + app).
2. Trigger ingestion for LAWA + MBIE sources using local ingestion endpoints.
3. Query:
   - LAWA assessments by indicator, trend window, direction, region/site
   - MBIE generation by fuel and date range
   - Renewable share by quarter
4. Call an insights endpoint that returns derived summaries **plus a fact pack** (sources, publish dates, import timestamps, caveats).

---

## Explicit non-goals (MVP)

- **No raw water monitoring time-series ingestion** (sensor/sample level feeds are out of scope).
- **No ML / forecasting / prediction.**
- **No public deployment** (local-only).
- **No auth/rbac** (assume trusted local use).
- **No UI/frontend** (backend + docs only).
- **No autonomous AI commits** (human-in-the-loop for all commits).

---

## MVP-critical requirements (only the essentials)

- **Idempotent ingestion**: re-ingesting identical content must not duplicate records.
- **Dataset lineage**: every persisted fact is traceable to a dataset release (source URL, published date, imported time).
- **Fact-pack-first**: any endpoint returning “insights” must include a fact pack; if a claim can’t be grounded in ingested data, it must be omitted or explicitly marked as unavailable.

---

## Success criteria

The MVP is successful when:

### Local run
- `docker compose up` (or equivalent) starts Postgres + the app locally without manual setup steps beyond documented environment variables.

### Ingestion
- LAWA + MBIE datasets can be ingested end-to-end into Postgres from public URLs.
- Re-running ingestion with the same file produces:
  - no duplicate facts
  - clear “no-op / already imported” behavior (or metadata-only update)
- Failures are diagnosable from logs and persisted release status.

### APIs
- APIs are versioned under `/api/v1`.
- List endpoints are paginated and filterable.
- “Overview/insights” endpoint:
  - returns derived summaries (counts, shares)
  - always includes fact pack: sources used, published dates, import timestamps, caveats.

### Quality bar (portfolio)
- Migrations exist for schema creation.
- Integration tests validate: ingest → query (happy path) and idempotent re-ingest.
- README explains: run, ingest, sample curl calls.

## Local-only ingestion safeguards (implementation note)

Although ingestion endpoints are exposed under `/api/v1/ingest/*`, they are **MVP-local only**.

**Required guard (implement at least one):**
- Enable ingestion endpoints **only under the `dev`/`local` Spring profile**, OR
- Require a simple static header token (e.g. `X-INGEST-TOKEN`) configured via environment variable.

**Rationale**
This prevents accidentally shipping open ingestion endpoints if the service is deployed later,
while preserving simple local workflows for the MVP.

This constraint is mandatory for MVP completeness.
